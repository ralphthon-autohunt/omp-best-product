# Validator 에이전트 개선 제안

## 배경

현재 Validator는 Sean Ellis 4질문을 기반으로 PMF Score를 산출하고 GRADUATE/ITERATE/PIVOT/KILL을 판정한다. 기본 프레임은 맞지만, **설문 결과를 제품 개선에 연결하는 과정**에서 핵심 로직이 빠져 있다.

Superhuman이 PMF 22% → 58%로 끌어올린 과정, 그리고 alwayz(레브잇)가 실전에서 검증한 방법론을 반영하면, ITERATE 루프의 정밀도를 크게 높일 수 있다.

### 레퍼런스

- [초기 스타트업을 위한 PMF 서베이 가이드 (leansprint.kr)](https://leansprint.kr/pmf-survey-guide-for-early-startups/)
- [PMF를 과학적으로 추적하는 법 (alwayz / 레브잇)](https://blog.alwayz.co/pmf)

## 변경 1: SD 세분화 (VSD / NSD)

### 현재

Q1 응답을 VD / SD / ND / N/A 4개로 분류하고 끝.

### 변경 후

SD를 한 번 더 쪼갠다.

| 분류 | 정의 | 역할 |
|------|------|------|
| **VD** | 매우 실망 | 핵심 가치의 증거. 이 사람들이 좋아하는 것을 강화 |
| **VSD** | 약간 실망 + 핵심 가치에 공감 | VD로 전환 가능한 잠재 팬. 이 사람들의 부족한 점만 보완 |
| **NSD** | 약간 실망 + 핵심 가치에 미공감 | 타겟이 아님. 피드백 무시 |
| **ND** | 실망 안 함 | 타겟이 아님. 피드백 무시 |

### VSD/NSD 구분 방법

Q3(핵심 혜택) 응답이 VD 그룹의 Q3 Top 키워드와 겹치면 VSD, 안 겹치면 NSD.

### 왜 유의미한가

SD는 전체 응답의 30~40%를 차지하는 가장 큰 그룹이다. 이 안에서 "거의 팬인 사람"과 "애초에 타겟이 아닌 사람"을 구분하지 않으면, 제품 방향이 양쪽으로 찢어진다. VSD만 골라내면 **가장 적은 노력으로 VD 비율을 올릴 수 있는 개선점**이 보인다.

## 변경 2: 피드백 필터링

### 현재

Deep 10 + Mid 15 전원의 Q3/Q4를 동등하게 집계 → Top 키워드 추출 → Builder-PM에 전달.

### 변경 후

응답자 분류에 따라 피드백 가중치를 다르게 적용한다.

| 응답자 | Q3 (강점) | Q4 (개선점) |
|--------|-----------|------------|
| **VD** | 핵심 가치 정의에 사용 | 강화 방향 설정에 사용 |
| **VSD** | 참고만 | **VD 전환용 기능 도출에 사용** |
| **NSD** | 배제 | 배제 |
| **ND** | 배제 | 배제 |

### 왜 유의미한가

ND가 "소셜 기능 넣어주세요"라고 하면 현재 구조에서는 Top 키워드에 올라갈 수 있다. 이걸 반영하면 핵심 가치가 희석되고, VD였던 사람들마저 이탈한다. **ND/NSD의 목소리를 명시적으로 배제**하는 것이 핵심 가치를 지키는 가장 확실한 방법이다.

## 변경 3: 이터레이션 목표 재정의

### 현재

PMF Score를 올리는 것이 목표. Q4 개선점을 전체 반영.

### 변경 후

2-track으로 분리한다.

```
Track A (핵심 강화): VD가 좋아하는 것을 더 좋게 만든다
  → 입력: VD의 Q3 + Q4
  → 효과: 기존 VD 유지, 이탈 방지

Track B (전환 유도): VSD가 부족하다고 느끼는 것을 채운다
  → 입력: VSD의 Q4만
  → 효과: VSD → VD 전환, PMF Score 상승
```

### 왜 유의미한가

PMF Score를 올리는 경로는 두 가지뿐이다 — VD를 지키거나, 새로운 VD를 만들거나. Track A가 전자, Track B가 후자다. 이 구분 없이 "전체 개선점 Top 3"를 반영하면 둘 다 놓친다.

## 변경 4: 추세 추적

### 현재

매 스프린트 PMF Score 단일 스냅샷 → 임계값 기반 판정 (≥55 GRADUATE 등).

### 변경 후

스프린트별 추세를 기록하고 판정에 반영한다.

```json
{
  "sprint_history": [
    { "sprint": 1, "vd_ratio": 0.22, "vsd_ratio": 0.18, "segment": "전체" },
    { "sprint": 2, "vd_ratio": 0.28, "vsd_ratio": 0.15, "segment": "전체" },
    { "sprint": 2, "vd_ratio": 0.33, "vsd_ratio": 0.12, "segment": "HXC" }
  ]
}
```

판정 보조 기준:

- 3스프린트 연속 상승 중이면 ITERATE 유지 (PIVOT/KILL 유예)
- 3스프린트 연속 정체면 PIVOT 검토 촉진

### 왜 유의미한가

PMF는 "달성했다/못했다"의 이분법이 아니라 **진전의 방향과 속도**가 중요하다. 22% → 28% → 33%면 아직 40% 미만이지만 명백히 올바른 방향이다. 현재 구조에서는 이걸 PIVOT/KILL로 판정할 수 있다.

## 변경 5: 세그먼트별 PMF 측정

### 현재

50명 전체를 한 덩어리로 PMF Score 계산.

### 변경 후

전체 + HXC 세그먼트별 별도 계산.

```
전체 PMF:         22% (KILL 영역)
HXC 세그먼트만:   33% (ITERATE 영역) ← 이쪽이 진짜 신호
```

### 왜 유의미한가

Superhuman 사례가 정확히 이것이다. 전체로 보면 22%였지만, "빠른 이메일을 원하는 프리랜서/창업자"만 떼어내면 33%였다. 거기에 집중해서 58%까지 올렸다. **모든 사람을 위한 제품은 아무도 위한 제품이 아니다.** 세그먼트별 측정이 있어야 "누구를 위한 제품인가"가 데이터로 보인다.

---

## 변경 6: 판정 기준 조정

### 현재

| 점수 | 판정 |
|------|------|
| ≥ 55 | GRADUATE |
| 40~54 | ITERATE |
| 25~39 | PIVOT |
| < 25 | KILL |

### 변경 후

| 점수 | 판정 | 근거 |
|------|------|------|
| ≥ 40 | GRADUATE | Sean Ellis 원본 기준. 실리콘밸리 표준 |
| 25~39 | ITERATE | 가능성 있음. 세분화 + 집중으로 40% 도달 가능 |
| 15~24 | PIVOT | 현재 세그먼트에서는 어려움 |
| < 15 | KILL | 근본적 수요 부재 |

### 왜 유의미한가

55%는 Sean Ellis 원본 어디에도 없는 기준이다. 40%가 수백 개 스타트업에서 경험적으로 검증된 임계값이고, 이 기준을 높이면 실제로는 PMF를 달성한 제품을 불필요하게 ITERATE 시키게 된다.

## 전체 흐름 변화 요약

### 현재

```
설문 → 전체 집계 → 스냅샷 판정 → 전체 피드백 반영 → 재빌드
```

### 변경 후

```
설문 → SD 세분화 → 세그먼트별 측정 → 추세 반영 판정
                                          │
                              VD 피드백 → Track A (강화)
                              VSD 피드백 → Track B (전환)
                              ND/NSD → 배제
                                          │
                                      재빌드 → 재측정
```

핵심 차이는 **"무엇을 만들까"의 입력이 달라진다**는 것이다. 현재는 50명의 평균적인 목소리가 입력이고, 변경 후에는 **핵심 팬(VD)과 잠재 팬(VSD)의 목소리만** 입력이 된다.
